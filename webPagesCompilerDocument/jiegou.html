基本结构

 我们的webPages文档分为5个模块，缺一不可。
  1. DOCTYPE
  2. set
  3. template
  4. style
  5. script

DOCTYPE

像HTML一样，DOCTYPE属于文档的声明。没有了DOCTYPE，等于webPages文档就没有了身份证号，寸步难行。

如何进行声明呢？

HTML中的声明

这里说HTML5。很简单： 
<!DOCTYPE html>

实际上，webPages也一样简单：

<!DOCTYPE webPages version="1.0">

不像plist拥有繁琐的声明过程，只是和HTML一样的DOCTYPE，只不过添加了一个版本，其实更加方便。比如现在的HTML4，就需要其他的文件进行引入，而我们的webPages结构升级后，Compiler会保留老版本的解释，淘汰时可以安装老版本。这样，就一点也不麻烦了。

注：webPages文档的DOCTYPE只写一行且没有多余的空行。

set

实际上， set元素就是HTML中head除了文件内的CSS(<style>)的所有元素。我们虽然去除了style元素，但是使得开发更加有条理性，虽然加多了Compiler的编写，但是为了我们能够更好地使用。

基本的set：

<set>
	<meta charset="utf-8"></meta>
	<title></title>
</set>
怎么样，是不是很简单呢！ 同时，我们还可以引入各式各样的文件，比如： 
<set>
	<script src="https://unpkg.com/vue@2.6.11/dist/vue.js" type="text/javascript" charset="utf-8"></script>
</set>
这样就可以在webPages中把Vue.js引入了。

template

其实，template模块就是堂堂正正的HTML中的主干部分。
为什么呢？为什么，请问您的英语老师吧。我喜欢叫那个HTML页面为HTML板块，template其实就是板块的意思。

在<template>模块中，我们像上文一样随心所欲地编写网页内容。不用愁什么不对应，这个，全部包在webPages Compiler的身上！

比如，我们编写一个Hello, World!

<template>
	<h1>Hello, World!</h1>
</template>

通过我们webPages Compiler加上<!DOCTYPE webPages version="1.0">，<set>，<style>和<script>的编译转换，直接在对应的地方输出了这样：

<!DOCTYPE html>
<html>
	<head>
		<meta charset="utf-8"></meta>
		<title></title>
		<style type="text/css">
			
		</style>
	</head>
	<body>
		<h1>Hello, World!</h1>
		<script type="text/javascript">
			
		</script>
	</body>
</html>

加粗部分就是template中的内容。

注：如果在里面引入Vue.js并且使用var vm=new Vue();引入，请自带<div id="app">

style

<style>元素就是CSS的直接表述。这里(webPages Document 1.0)只是CSS，以后会加入Less或者Sass。

我们做一个将class为.center的元素放置到中间

<style>
	.center{
		text-align: center;
	}
</style>

由于安全起见，<style type="text/css">是编译以后的成果。

<style type="text/css">
	.center{
		text-align: center;
	}
</style>

script

<script>是HTML中的脚本模块。

譬如说：我们想要把Vue.js通过var vm=new Vue();引入，并且绑定了el, data和methods：

<script>
	var vm=new Vue({
		el:"#app",
		data(){
			return{
				
			}
		},
		methods:{
			
		}
	});
</script>

经过webPages Compiler的编译，就变成了：

<script type="text/javascript">
	var vm=new Vue({
		el:"#app",
		data(){
			return{
				
			}
		},
		methods:{
			
		}
	});
</script>

同<style>一样，我们在里面追加了text="javascript"属性。
在以后，我们会加入typescript，但不是<script type="ts">
